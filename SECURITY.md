## 2. Реализованные защиты

### 2.1. Content Security Policy (CSP)

`index.html` имеет строгую CSP:

```
default-src 'self';
script-src 'self' 'unsafe-inline';
style-src 'self' 'unsafe-inline';
img-src 'self' data: blob:;
font-src 'self' data:;
connect-src 'self' https://api.groq.com https://openrouter.ai https://api.anthropic.com http://localhost:* https://localhost:* http://127.0.0.1:* https://127.0.0.1:*;
manifest-src 'self';
worker-src 'self';
base-uri 'self';
form-action 'none';
object-src 'none';
```

**Что блокируется:**
- ❌ Загрузка скриптов с внешних доменов (нет CDN)
- ❌ `eval()`, `new Function()` (Function-объект через CSP запрещён в worker)
- ❌ `<object>`, `<embed>`, Flash-плагины
- ❌ Подача форм на любой URL (no `<form action>`)
- ❌ Изменение `<base>` для подмены ссылок

### 2.2. Защита от XSS

Все пользовательские данные проходят через `escapeHtml()`:

```javascript
function escapeHtml(s) {
  return String(s)
    .replace(/&/g,'&amp;')
    .replace(/</g,'&lt;')
    .replace(/>/g,'&gt;')
    .replace(/"/g,'&quot;')
    .replace(/'/g,'&#39;');
}
```

`renderMd()` экранирует HTML **до** применения markdown — невозможно инжектировать `<script>` через `**`/`` ``` ``.

### 2.3. Защита от clickjacking

Frame-busting в начале JS:

```javascript
if (window.top !== window.self) {
  // Блокируем работу + показываем предупреждение
  document.body.innerHTML = '⚠️ GrandHubAi не может работать во встроенном фрейме';
  throw new Error('Clickjacking attempt blocked');
}
```

Дополнительно: при HTTPS-деплое сервер должен отправлять заголовок `X-Frame-Options: DENY`.

### 2.4. Шифрование API-ключа

Опционально (Settings → Безопасность → ☑ Шифровать API-ключ master-паролем):

- **Алгоритм:** AES-256-GCM
- **Производный ключ:** PBKDF2 с SHA-256, 100 000 итераций
- **Соль:** 16 байт случайности (per-key)
- **IV:** 12 байт случайности (per-encryption)
- **Web Crypto API** — стандартная браузерная криптография

Без правильного пароля API-ключ восстановить **невозможно**.

### 2.5. Изоляция данных

- **IndexedDB** — origin-isolated, доступ только сайту с того же origin
- **Без cookies** — нет передачи сессии в сторонние запросы
- **Без serverside** — у нас нет своего бэкенда, нет точки утечки на нашей стороне
- **Network-only для API** — Service Worker не кэширует ответы LLM

### 2.6. Прямая связь с провайдером

API-ключ отправляется **только** на хост соответствующего провайдера:
- Groq → `api.groq.com`
- Anthropic → `api.anthropic.com`
- OpenRouter → `openrouter.ai`
- Ollama → `localhost:11434`

CSP `connect-src` явно перечисляет разрешённые хосты — даже скомпрометированный JS не сможет отправить ключ на чужой сервер.

### 2.7. Anti-prototype-pollution

Все парсеры JSON используют `JSON.parse()` — он не уязвим к prototype pollution.
Объект `state.settings` определён литерально, нет `Object.assign` без валидации полей.

### 2.8. Валидация ввода

- `state.settings.theme` валидируется по `VALID_THEMES`
- `state.settings.fontSize` валидируется по `VALID_FONT_SIZES`
- `state.settings.orchestratorScale` приводится к `parseInt`, fallback 3
- Endpoint URL валидируется через `new URL()` — невалидные значения отбрасываются
- API-ключ проходит `trim()` (нельзя случайно подсунуть пробельный prefix)

### 2.9. Безопасный экспорт

При экспорте JSON API-ключ автоматически заменяется на `[REDACTED]`. Нельзя случайно поделиться файлом, содержащим ваш ключ.

### 2.10. Заголовки безопасности (при деплое)

При деплое на сервер (Vercel, Cloudflare, nginx) добавьте заголовки:

```
Strict-Transport-Security: max-age=31536000; includeSubDomains
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: camera=(), microphone=(self), geolocation=()
```

Готовый `start.py` уже добавляет основные из них при локальном запуске.

## 3. Что защитить нельзя

### 3.1. Malware на пользовательском ПК

Если на компьютере установлен keylogger / RAT / infostealer — никакая клиент-side защита не поможет:
- Браузерные расширения с правом «Read all data on visited sites» имеют полный доступ к IndexedDB
- Скриншоты экрана покажут master-пароль при вводе
- Браузерные dump-tools скопируют профиль целиком

**Рекомендация:** установите EDR (Windows Defender актуальный, Kaspersky, ESET); не ставьте подозрительные расширения; регулярно проверяйте `chrome://extensions/`.

### 3.2. Физический доступ

Если кто-то имеет физический доступ к разблокированному компьютеру — у него есть всё.

**Рекомендация:** блокируйте экран (Win+L); шифруйте диск (BitLocker, FileVault); используйте сильный пароль входа.

### 3.3. Кража кода через Git/HTTP

Весь код **открыт** под лицензией AGPL-3.0. Любой может:
- Скачать
- Прочитать промпты
- Сделать форк

**Это by design.** Защита здесь не техническая, а правовая:
- AGPL-3.0 обязывает форки публиковать модификации
- Использование в закрытом продукте требует Commercial License
- Бренд защищён ТЗ
- При обнаружении нарушений → юрист → суд

### 3.4. Атаки на провайдеров LLM

Если Groq / Anthropic / OpenRouter скомпрометированы — мы не контролируем это.

**Рекомендация:** используйте API-ключи с минимальными правами и rate-лимитами; мониторьте usage; не используйте один ключ для production и тестов.

## 4. Рекомендации пользователю

### 4.1. Управление API-ключами

✅ **Да:**
- Создайте отдельный API-ключ для GrandHubAi
- Установите rate-limit на стороне провайдера (Groq → Usage → Set limit)
- Включите шифрование master-паролем (Settings → Безопасность)
- Регулярно ротируйте ключи (раз в 3 месяца)

❌ **Нет:**
- Не используйте один ключ для production-приложения и GrandHubAi
- Не делитесь ключом
- Не коммитьте ключ в Git
- Не вставляйте ключ через копирование с публичных скриншотов

### 4.2. Master-пароль

✅ **Да:**
- 12+ символов, разные регистры, цифры, символы
- Запишите в менеджер паролей (1Password, Bitwarden, KeePass)

❌ **Нет:**
- Не используйте тот же пароль что для почты
- Не записывайте на стикере на мониторе
- При утере **восстановить нельзя** — придётся вводить ключ заново

### 4.3. Обновления

- Регулярно обновляйте версию GrandHubAi (`git pull` или скачивайте с офиц. сайта)
- Проверяйте `CHANGELOG.md` на security-исправления
- Подписка на security-объявления: см. п. 5

### 4.4. Подозрительная активность

Симптомы компрометации:
- API-провайдер показывает запросы которые вы не делали
- Внезапно опустошился free tier
- Появились странные сообщения в чате
- Тема/настройки самопроизвольно поменялись

Действия при подозрении:
1. Немедленно отозвать API-ключ у провайдера
2. Settings → 🗑️ Очистить всю историю и настройки
3. Проверить расширения браузера
4. Сменить master-пароль (если использовался)
5. Прислать отчёт на aslankaa@yandex.ru

### 6.3. Технические меры защиты

#### Checksum верификация (планируется в v1.2)

Каждый файл промптов будет иметь SHA-256 хэш в манифесте:

```javascript
const PROMPT_INTEGRITY = {
  "coordination.js": "sha256-AbCd...",
  "engineering.js": "sha256-EfGh...",
};
```

При загрузке в браузере проверяется хэш — несовпадение = модифицированная версия.

#### Watermark в промптах

Скрытые маркеры в системных промптах ("сгенерировано GrandHubAi для пользователя X на дату Y"). Это позволяет идентифицировать утечки в публикациях.

#### Code signing (для desktop-версии)

Когда выйдет Tauri-версия — будет подписана сертификатом организации (Authenticode для Windows, Apple Developer для macOS). Подделка не пройдёт проверку SmartScreen / Gatekeeper.
