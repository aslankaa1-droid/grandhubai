# 📘 GrandHubAi — Руководство пользователя

**Для версии:** 1.0.0
**Уровень:** базовый — никакого программирования не требуется

---

## 🚀 Шаг 1. Запуск приложения

### Если у вас просто папка с файлами

1. Откройте папку `GrandHubAi/src/`
2. Двойной клик по файлу `index.html`
3. Откроется в вашем браузере по умолчанию

### Если хотите установить как приложение на смартфон/ПК

1. Откройте `index.html` через локальный сервер (см. ниже) или с домена
2. В Chrome/Edge нажмите ⊕ в адресной строке → «Установить GrandHubAi»
3. Появится иконка на главном экране (телефон) или в меню Пуск (Windows / macOS Dock)

**Запуск локального сервера** (один раз):
```
Windows + Python:    cd src && python -m http.server 8080
Windows + Node.js:   cd src && npx http-server -p 8080
macOS:               cd src && python3 -m http.server 8080
```
Затем откройте `http://localhost:8080`.

## 🔑 Шаг 2. Получение бесплатного API-ключа Groq (рекомендуется)

GrandHubAi сам по себе не имеет встроенного ИИ — он подключается к внешним моделям. **Groq** — самый быстрый и бесплатный вариант.

1. Зайдите на [console.groq.com](https://console.groq.com)
2. Зарегистрируйтесь (Google/GitHub/email — 1 минута)
3. В левом меню → **API Keys** → **Create API Key**
4. Скопируйте ключ (начинается с `gsk_...`)

> ⚠️ Сохраните ключ в надёжном месте — Groq показывает его только один раз. При утере — создаёте новый.

**Альтернативные провайдеры** (если Groq не подходит):
- **OpenRouter** ([openrouter.ai](https://openrouter.ai)) — даёт доступ к Claude, GPT, Gemini, Llama одним ключом. Платный по факту использования (~$1-5/мес для среднего пользователя)
- **Anthropic** ([console.anthropic.com](https://console.anthropic.com)) — Claude напрямую. Премиум-качество, $3-15 за 1М токенов
- **Ollama** ([ollama.com](https://ollama.com)) — установка локально на ваш ПК. Полностью бесплатно, без интернета

## ⚙️ Шаг 3. Настройка GrandHubAi

1. В правом верхнем углу нажмите ⚙️ **Настройки**
2. **Провайдер:** выберите Groq
3. **API ключ:** вставьте ваш `gsk_...`
4. **Endpoint:** оставьте пустым (используется значение по умолчанию)
5. **Модель по умолчанию:** оставьте `llama-3.3-70b-versatile` (или выберите другую из подсказки)
6. **Размер команды агентов на задачу:** 3 (рекомендуется для начала)
7. **Streaming-режим:** ✅ включён
8. Нажмите **Сохранить**

В верхнем правом углу зелёный индикатор должен сменить надпись на «Groq активен». Готово.

## 💬 Шаг 4. Первый запрос

Введите задачу в нижнее поле и нажмите 📤. Например:

> *"Спроектируй конструкцию насоса для перекачки воды Q=50 м³/ч на высоту 40 м. Дай 3D-модель и чертёж."*

Что произойдёт:
1. **Oracle** разобьёт задачу на подзадачи
2. **Команда специалистов** запустится параллельно:
   - Инженер насосного оборудования → подбор насоса
   - 3D-моделлер → последовательность построения в SolidWorks
   - Инженер-чертёжник ЕСКД → правила оформления чертежа
3. **Oracle** соберёт ответы в единый текст

В **Activity Log** (📡 справа) видно, кто из специалистов сейчас работает.

## 🤖 Шаг 5. Просмотр библиотеки агентов

Слева — иерархия из 201 агента в 27 доменах.

- **Поиск:** введите ключевое слово в поле «🔍 Поиск агента» — появятся подходящие
- **Фильтр по домену:** в верхнем меню выберите домен (например, «Медицина»)
- **Просмотр промпта:** клик по агенту → откроется модальное окно с системным промптом, температурой, примерами
- **Прямой вызов:** в окне агента нажмите **«Прямой вызов»** → ваш следующий запрос пойдёт напрямую к этому агенту, минуя оркестратора

1. Кликните 🎤

## 🛟 Решение проблем

### «API-ключ не задан»
→ Откройте ⚙️ Настройки и впишите ключ.

### «HTTP 401: Unauthorized»
→ Ключ неверный или отозванный. Создайте новый в личном кабинете провайдера.

### «HTTP 429: Too Many Requests»
→ Превышен лимит провайдера. На Groq дневной лимит ~14400 токенов/мин. Подождите минуту или переключитесь на другого провайдера.

### «HTTP 400: Bad Request»
→ Скорее всего, неверная модель. Зайдите в настройки и выберите модель из выпадающего списка (например, `llama-3.3-70b-versatile`).

### Голос не работает
→ Используйте Chrome или Edge. В Yandex Browser работает с встроенным ассистентом.

### PWA не устанавливается
→ Нужен HTTPS или localhost. Через `file://` PWA-функции отключены — это нормально.

### Истории не сохраняются между сеансами
→ Проверьте настройки приватности браузера: данные IndexedDB могут чиститься в режиме «Инкогнито» при закрытии.

### Хочу очистить всё
→ ⚙️ Настройки → Очистить всю историю и настройки.

## 💡 Советы для эффективной работы

1. **Уточняйте контекст в задаче.** Чем детальнее формулировка, тем точнее декомпозиция Oracle. «Расчитай вал» — плохо. «Расчитай вал ⌀40 мм длина 600 мм с двумя подшипниками, передающий момент 250 Н·м, материал сталь 40Х» — хорошо.

2. **Ставьте размер команды по сложности задачи:**
   - **1** — прямой вопрос с одним ответом
   - **3** — большинство задач (по умолчанию)
   - **5-7** — комплексные мульти-доменные задачи
   - **10** — для экспериментов / финальных отчётов

3. **Используйте «Прямой вызов» агента,** когда точно знаете, кто вам нужен — экономит вызовы и время.

4. **Сохраняйте важные диалоги** через 📝 MD сразу после получения ответа — иначе IndexedDB может перезаписать через 200 сообщений.

5. **Голосом проще для длинных задач** — диктуете быстрее, чем печатаете.
