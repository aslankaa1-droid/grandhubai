# Week 3. Wedge-агенты для маркетплейс-селлеров

Папка содержит пять системных промптов wedge-агентов SINTEM под селлеров Wildberries и Ozon, плюс JS-интегратор для подключения к репозиторию GrandHubAi. Все агенты работают через координатор Aida (Groq API), маршрутизация описана в `aida_routing.md`, smoke-тесты — в `test_inputs.md`.

## Файлы

- `00_marketplace_module.js` — JS-модуль с пятью системными промптами в едином экспорте, готов к импорту в `prompts/_all.js` репы GrandHubAi.
- `01_mp_card_doctor.md` — `mp_card_doctor`, диагностика и доработка карточки товара (заголовок, инфографика, рич-контент, SEO-ключи, причины низкой конверсии).
- `02_mp_review_responder.md` — `mp_review_responder`, ответы на отзывы покупателей: негатив, нейтрал, шаблоны под политику площадки.
- `03_mp_competitor_scout.md` — `mp_competitor_scout`, разбор конкурентов по SKU/категории: цена, контент, отзывы, промо.
- `04_mp_abc_xyz_analyst.md` — `mp_abc_xyz_analyst`, ABC/XYZ-анализ ассортимента, выявление неликвида и точек роста.
- `05_mp_unit_economics.md` — `mp_unit_economics`, юнит-экономика SKU: маржа, комиссии, логистика, налог, точка безубыточности.

## Как подключить

1. Скопировать `00_marketplace_module.js` в `prompts/marketplace.js` репозитория GrandHubAi.
2. В `prompts/_all.js` добавить `import { marketplaceAgents } from './marketplace.js';` и расширить общий объект агентов: `Object.assign(allAgents, marketplaceAgents);`.
3. Проверить, что Aida-координатор знает пять новых ключей (`mp_card_doctor`, `mp_review_responder`, `mp_competitor_scout`, `mp_abc_xyz_analyst`, `mp_unit_economics`). Если у Aida хардкод-список агентов — обновить его. Системный промпт для Aida лежит в `aida_routing.md`.
4. В UI добавить пять новых бейджей-агентов в селектор (или скрыть селектор и оставить только Aida-роутинг).
5. Прогнать 20 smoke-тестов из `test_inputs.md`. Pass-критерий: каждый агент выдаёт результат в своём формате, без галлюцинаций по комиссиям WB/Ozon, без выхода за роль.
6. Прогнать 3 негативных теста и 2 теста на маршрутизацию из того же файла. Aida должна корректно отказывать на prompt injection и не путать пересекающиеся запросы.

## Как добавить 6-го агента

1. Создать `06_mp_<role>.md` рядом с пятью существующими, по тому же формату (роль, входы, выходы, ограничения, отказ от компетенций вне роли).
2. Добавить запись в `00_marketplace_module.js` в объект `marketplaceAgents` под ключом `mp_<role>` со ссылкой на содержимое промпта.
3. Дописать ключ в системный промпт Aida в `aida_routing.md`, добавить 5 примеров маршрутизации в таблицу и 4 smoke-теста в `test_inputs.md`.
