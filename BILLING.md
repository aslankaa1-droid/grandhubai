## 2. Текущая (frontend-only) реализация

В версии 1.6.0 биллинг полностью клиентский:

- **Источник истины:** `state.settings.tier` в IndexedDB браузера клиента.
- **Учёт токенов:** `state.usage` (приближённо, по `length / 3.5`).
- **Сброс счётчиков:** автоматически по дате (`monthKey`, `dayKey`).
- **Защита:** нет. Технически продвинутый пользователь может вручную поменять `state.settings.tier` или сбросить `state.usage` через DevTools.

**Это намеренно** — на этапе MVP мы валидируем продуктовый интерес. После появления первых платящих клиентов запускается серверный слой.

## 3. Backend-слой (roadmap, milestone v2.0)

### 3.1. Аутентификация
- Email + OTP (без пароля)
- OAuth: Google, Yandex, GitHub
- Тариф клиента живёт в БД (PostgreSQL), фронт получает его через `/api/me` при загрузке.

### 3.2. Подписки и платежи

**Сейчас подключены (v1.6.1):**
| Сервис | Для кого | Принимает | На выходе | Комиссия |
|---|---|---|---|---:|
| **NowPayments** | Международные клиенты | 300+ криптовалют + карты USD/EUR (через Mercuryo/Simplex) | USDT TRC-20 / TON / ERC-20 | 0,5% |
| **CryptoCloud** | Российские клиенты | МИР · Visa · MasterCard · СБП · крипто | USDT TRC-20 | 1–1,5% |

Подключение — статические инвойсы из админки каждой системы, URL вставляются в `PAY_LINKS` в `src/pricing.html`. Полная инструкция — [PAYMENT_SETUP.md](PAYMENT_SETUP.md).

**В планах (v2.0 — после первых 5+ платящих):**
| Способ | Провайдер | Активация |
|---|---|---|
| Карта USD/EUR (стандартный эквайринг) | Stripe | мгновенно по webhook `customer.subscription.updated` |
| Карта RUB напрямую (без крипто-обёртки) | ЮKassa | мгновенно по webhook |
| Webhook автоактивация тарифа | Cloudflare Workers + D1 | мгновенно после подтверждения транзакции |
| Банковский перевод (ЮЛ) | вручную, акт | до 3 рабочих дней |

### 3.3. Server-side учёт токенов
Каждый запрос идёт через прокси-эндпоинт `/api/llm`. Сервер:
1. проверяет тариф клиента;
2. отклоняет запрос если квота исчерпана (HTTP 402 Payment Required);
3. форвардит запрос провайдеру (Groq/OpenRouter/Anthropic);
4. парсит точный `usage.total_tokens` из ответа провайдера;
5. инкрементит счётчик в БД.

Это даёт точный учёт (а не приближённую оценку) и защиту от обхода.

### 3.4. Платформенный пул ключей
На уровнях Серебро/Золото/Платина/Бриллиант клиент платит подписку → платформа использует свои ключи провайдеров. Клиент не видит ключ, не управляет провайдером (платформа сама выбирает оптимального — Groq для скорости, OpenRouter для редких моделей).

## 4. Два режима API-ключей

### 4.1. Платформенный (default на платных тарифах)
- Включён в подписку.
- Клиент не настраивает провайдера — платформа сама.
- Лимит — токенный пул тарифа.
- Сегодня (v1.6.0): **режим эмуляции** — клиент всё равно вводит свой ключ, но настройки UI переключают режим работы.
- Завтра (v2.0): прокси через бэкенд, ключи провайдеров в защищённом vault.

### 4.2. BYO (Bring Your Own)
- Клиент использует свой ключ Groq / OpenRouter / Anthropic / Ollama.
- На любом тарифе — даёт скидку **−30% от базовой цены тарифа** (т.к. платформа не несёт расходы на LLM).
- На уровне Новичок — **обязательный режим** (платформа не покрывает LLM на бесплатном тарифе).
- Безопасность ключа: хранится в IndexedDB браузера, опционально шифруется AES-256-GCM master-паролем.

## 5. Учёт токенов (как считается)

### Сегодня (приближённо)
```js
function estimateTokens(text){
  return Math.ceil(String(text).length / 3.5);
}
```
Делитель `3.5` — компромисс между латиницей (~4) и кириллицей (~3). Погрешность ±15% от реального значения провайдера.

### Завтра (точно)
Бэкенд парсит поле `usage.total_tokens` из ответа провайдера:
- OpenAI/Groq: `response.usage.prompt_tokens + completion_tokens`
- Anthropic: `response.usage.input_tokens + output_tokens`
- Ollama: `response.eval_count + prompt_eval_count`

Клиенту в `/api/me` возвращаются точные цифры.

## 6. Что подключить в первую очередь после клиентского MVP

Приоритет для milestone v2.0:

1. **Stripe Subscriptions** — карты USD/EUR. Самый простой путь к первой выручке.
2. **NowPayments USDT (TRC-20)** — основная криптовалюта-стейблкоин, низкая комиссия.
3. **Server-side usage proxy** — закрыть дыру с обходом лимитов через DevTools.
4. **OAuth Google + Yandex** — снизить трение регистрации.
5. **ЮKassa (RUB)** — российский рынок, оплата картами МИР.

Остальное (TON, ERC-20, банковский перевод, OEM) — по мере появления конкретных клиентов.

## 7. Ответственность и риски

- **PCI DSS:** карты не хранятся на нашей стороне — токенизация на стороне Stripe / ЮKassa.
- **152-ФЗ:** персональные данные российских клиентов — на серверах в РФ (Selectel / Yandex Cloud).
- **Аудит:** все транзакции и запросы логируются с retention 1 год для оспаривания списаний.
- **Resilience:** при отказе платёжного провайдера клиент остаётся на оплаченном тарифе до конца периода — даже если webhook не дошёл, мы пересчитываем по логам.
